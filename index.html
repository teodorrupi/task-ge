3. SQL

a) From Employee ( id, name, department_fk ) (10 records) , Department ( id, name) (5 records) tables, selecting everything would return 50 results. Since we neglect to define the correlation between tables using a WHERE clause, this acts as in a CROSS JOIN between tables, giving the value of one column/-set associated with every other possible result selected. In this case SELECT * FROM 'Employee', 'Department' is equivalent to SELECT * FROM 'Employee' CROSS JOIN 'Department'

b) In order to select the first 5 records from a table, we can use limit on the results.
SELECT * from table_name limit 5

c) The Having clause its mostly useful with aggregate functions like COUNT, SUM, and is used with the Group By clause. The Group By clause is used to create groups of result-sets.
For example, table Spending with column:data
Name: 'A', 'B', 'C', 'A', 'B' and Spent: 10, 20, 15, 10, 10 is created.
SELECT Name, sum(Spent) from Spending group by Name;
This will allow us to treat each name as a group and have all non repeating names with their sums aggregated.
SELECT Name, sum(Spent) from Spending group by Name having sum(Spent) > 15;
This will allow us that from all the groups aggreated with their sum, to only show the ones which are Having CONDITION (here smaller than 15). In this case we exclude C from the result set.
